## Project

voidgun-gbc — A Game Boy Color remake of Voidgun (a multiplayer 2D space combat game). The original game (at C:/dev/spacedout) is a Java/LibGDX networked shooter with 3 ship classes, 10+ weapons, and FFA/TDM/CTF modes. This GBC version adapts those mechanics to the Game Boy Color's constraints.

## Build

GBDK-2020 v4.5.0 at `C:/gbdk`. hUGEDriver at `hugedriver/`.

```
C:/gbdk/bin/lcc -Wa-l -Wl-m -Wl-j -Wm-yC -Ihugedriver/include -Wl-lhugedriver/gbdk/hUGEDriver.lib -o voidgun.gbc src/main.c src/ship.c src/ship2.c src/flag_spr.c src/font.c src/music.c
```

Current ROM: 32KB (25KB used), no banking. CGB-only.

## Architecture

- `src/main.c` — everything: game states, physics, AI, collision, CTF, rendering, HUD
- `src/ship.c`, `src/ship2.c` — Team Gamma/Delta ship metasprite data (generated by png2asset)
- `src/flag_spr.c` — Flag metasprite data (generated by png2asset)
- `src/font.c` — Bitmap font tileset+map (generated by png2asset from assets/font.png)
- `src/music.c` — hUGEDriver song data (generated by uge2source from assets/music.uge)

### Key systems in main.c
- **Scrolling**: 80x80 tile procedural map, 32x32 BG ring buffer, `is_wall_fast()` bitmap cache
- **Flow fields**: BFS from each base at startup, stored in `flow_to_base0/1[400]`
- **Bot AI**: 5 bots, flow field + greedy steering + rotational probe + stuck/wander escape
- **CTF**: Flag states (HOME/CARRIED/DROPPED), carrier tracking, capture scoring
- **Combat**: 3 weapons, 8-bullet pool, 65 HP, respawn timer
- **HUD**: Window layer at WY=136, dirty-tracked updates

## Sound — CRITICAL NOTES

### What works
- hUGEDriver music via `add_VBL(hUGE_dosound)` — simple, no banking, works perfectly
- Music plays on menu, continues through credits, restarts on game/victory transitions
- `NR51_REG = 0x00` mutes before DISPLAY_OFF transitions, `0xFF` unmutes after

### What FAILED (do NOT repeat these approaches)
- **ROM banking for SFX**: Adding 11 SFX .c files pushed ROM past 32KB, requiring 64KB with MBC1 (`-Wm-yo4 -Wm-yt1`). This caused cascading crashes:
  1. `_CODE` segment overflowed bank 1 boundary — functions got split across banks, crashed when called
  2. VBL interrupt handler (`music_vbl`) did `SWITCH_ROM()` to read banked song data. This corrupted ANY data reads happening in the main thread from bank 1 (font tiles, bg tiles, etc.)
  3. `add_VBL`/`remove_VBL` during screen transitions caused crashes
  4. Setting `cur_music_bank` to bank 2 at boot made ALL bank-1 code inaccessible
  5. Calling `hUGE_dosound()` before `hUGE_init()` crashed on uninitialized data

### How to add SFX safely (TODO)
The core problem: hUGEDriver SFX are full `hUGESong_t` objects (~7KB each). 11 SFX = ~77KB. Can't fit in 32KB.

**Option A — Direct register writes (RECOMMENDED)**:
Instead of hUGESong_t SFX files, write audio registers directly for simple effects:
```c
// Laser sound on CH1
NR10_REG = 0x1E; NR11_REG = 0x10; NR12_REG = 0xF3;
NR13_REG = 0x00; NR14_REG = 0x87;
```
- Tiny code (~10 bytes per SFX vs 7KB)
- No banking needed
- Mute music channel with `hUGE_mute_channel(HT_CH1, HT_CH_MUTE)`, play SFX, unmute after N frames
- Fits easily in 32KB ROM

**Option B — Proper banking (COMPLEX)**:
If full hUGESong_t SFX are required:
- ALL functions that call `SWITCH_ROM()` MUST be `NONBANKED` (in bank 0)
- `hUGE_dosound()` from the library may be in bank 1 — verify with map file before switching banks
- NEVER have a VBL handler that switches banks — it corrupts main thread reads
- Instead, call `hUGE_dosound()` manually from the game loop with bank switching
- Must verify `hUGE_dosound` is in bank 0 (check map file: if address < 0x4000, it's bank 0)

## Original Game Reference

The original Voidgun source is at `C:/dev/spacedout` (Java/LibGDX). Key design elements to adapt:
- **Ship classes:** Light (fast, low HP), Medium (balanced), Heavy (slow, high HP)
- **Weapons:** Various fire rates, bullet speeds, spread patterns, AOE
- **Game modes:** Free-for-all, Team Deathmatch, Capture the Flag
- **Physics:** Zero-gravity thrust-based movement with rotation
- **Resolution:** Original is 960x540; GBC is 160x144
